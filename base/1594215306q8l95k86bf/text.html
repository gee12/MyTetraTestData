Yes, using the am (activity manager) command:

am start --user 0 -n 
com.package.name/
com.package.name.ActivityName

Here the first part (com.package.name) is the package of the app, and the second part com.package.name.ActivityName is the activity class.

Example for chrome:

am start --user 0 -n com.android.chrome/com.google.android.apps.chrome.Main

Example for Termux itself (bring it to foreground when e.g. ssh:ing in to the device):

am start --user 0 -n com.termux/com.termux.app.TermuxActivity
